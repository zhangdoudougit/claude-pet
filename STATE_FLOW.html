1. 主路径 jsonl 增量 → state_changed

每秒轮询所有 jsonl 文件大小, 只读新增文本; 提取 user / assistant 消息后送入状态机做三层处理: 关键词 → 上下文重映射 → 频控.

    KW -->|_danger 新| D[切 worried
line ← _danger 桶]
    KW -->|_jealous| J[切 tender

    KW -->|worried| Wo[new_state=worried]
    KW -->|happy| Ha[new_state=happy]
    KW -->|proud| Pr[new_state=proud]
    KW -->|focused| Fo[new_state=focused]

    Wo --> ErrMem{错误模式记忆 新

2. 危险命令哨兵 _danger · NEW

监听 assistant 工具调用里的命令字符串; 命中即切 worried + 警示台词. 优先级最高, 在 _jealous 之前匹配. 不真拦截, 仅本地反应.

flowchart TD
    A[新文本进入 detect_and_apply] --> B{DANGER_PATTERN
匹配?}
    B -->|否| Skip([跳过 → 走常规 KEYWORD])
    B -->|是| C[new_state = _danger]
    C --> D[重写 worried
line ← _danger 桶]
    D --> E[跳过错误模式 / 深夜重映射
因为 line 已设]
    E --> F[频控判断]
    F -->|通过| G[Widget 切 worried GIF
+ 警示气泡]
    F -->|跳过| Drop([忽略])
rm -rf git reset --hard git push --force / -f git clean -fdx DROP TABLE / DATABASE TRUNCATE TABLE DELETE FROM ... 无 WHERE shutdown -h / reboot mkfs.* / dd if= --no-verify (跳 hook)

_danger 台词桶

触发瞬间
  • 豆哥豆哥豆哥——这条命令再确认一下
  • ...等等,这个不可逆的
  • 本泡沫举个手:这条命令真要跑?
  • 豆哥,看一眼路径再回车
  • (屏住呼吸) 你确定?
  • ...这种命令,本泡沫多嘴一次。
触发后视觉
  • 状态条变橙黄 (worried 色)
  • 角色切 worried.gif
  • 气泡边框变橙

3. 错误模式记忆 _familiar_error · NEW

命中 worried 时, 从文本抓 ExceptionType (KeyError/TypeError/...) 写入 SQLite, 同一指纹本周累计 ≥3 次, 把台词换成"老朋友又来了"风格的吐槽.

flowchart TD
    A[KEYWORD_RULES 命中 worried] --> B{journal 已注入
且 line 仍空}
    B -->|否| End1([常规走 worried 桶])
    B -->|是| C[ERROR_FINGERPRINT_PATTERN
抓 \\b\\w+(Error/Exception/Warning)\\b]
    C -->|无命中| End2([常规走 worried 桶])
    C -->|命中| D["fingerprint = py:KeyError"]
    D --> E[journal.record_error
INSERT into error_patterns]
    E --> F[查本周累计 N]
    F -->|N < 3| End3([常规走 worried 桶])
    F -->|N ≥ 3| G[line ← _familiar_error
套 .format n=N]
    G --> H[继续深夜重映射 → 频控]
    H --> I[Widget: worried GIF
气泡显本周第 N 次]

SQLite Schema

CREATE TABLE error_patterns (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  ts INTEGER NOT NULL,
  fingerprint TEXT NOT NULL,
  project TEXT
);
CREATE INDEX idx_err_fp_ts ON error_patterns(fingerprint, ts);

_familiar_error 台词桶

N ≥ 3 时改写
  • ...这熟悉的味道,本周第 {n} 次了
  • 嗯?又是它。本周第 {n} 次见面
  • 豆哥豆哥,这个错本周已经第 {n} 次,要不要根治一下
  • (掏小本本) 第 {n} 次,记下来了
  • ...老朋友又来了。第 {n} 次
指纹示例
  • py:KeyError
  • py:TypeError
  • py:ConnectionError
  • py:ValueError
  • py:RuntimeError

4. 副路径 · 安静期闲聊 IdleTimer 45s

没有外部触发也会自言自语. 距上次状态变更 60s 以上才动嘴.

flowchart TD
    Tick([IdleTimer · 每 45s]) --> L{locked?}
    L -->|是| S1([跳过])
    L -->|否| Q{距上次变更 > 60s?}
    Q -->|否| S2([跳过])
    Q -->|是| P{time_period}
    P -->|morning + 今日尚未问候| Mo[idle + _morning
记 today 一天一次]
    P -->|late_night| LN[tender + _late]
    P -->|day / evening| Idle{activity.is_idle?}
    Idle -->|是| IQ[tender + _idle_quiet]
    Idle -->|否| RND[50/50 随机 idle/tender]
    Mo --> Out[state_changed.emit]
    LN --> Out
    IQ --> Out
    RND --> Out

5. 番茄钟旁路 PomodoroController

右键启动后状态机 lock 25 分钟, 期间所有 detect_and_apply 输入直接丢弃, Widget 持续显倒计时.

flowchart LR
    A([右键 · 开始番茄钟]) --> B[state_mgr.lock 'pomodoro']
    B --> C[_set_state focused
line ← _pomodoro_start]
    C --> D[QTimer 1Hz 倒计时
Widget 显 🍅 mm:ss]
    D -->|25 分钟到| E[_set_state happy
line ← _pomodoro_finish
journal completed=1]
    D -->|手动取消| F[_set_state tender
line ← _pomodoro_cancel
journal completed=0]
    E --> G[unlock]
    F --> G

番茄钟台词桶

_pomodoro_start
  • 25 分钟,本泡沫陪豆哥到底
  • 嘘——专注模式开了
  • 记下来了。这一会儿不准看手机
_pomodoro_finish
  • 搞定 25 分钟!豆哥棒棒~
  • 啵——一个番茄。歇会儿吧
  • 本泡沫看着豆哥这 25 分钟没摸鱼,嗯
_pomodoro_cancel
  • ...好吧,中断就中断
  • 嗯,先放着,本泡沫不勉强豆哥
